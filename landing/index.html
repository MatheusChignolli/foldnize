Features

Everything you need.
Nothing you don't.

Three renaming modes

Add a date prefix, fully replace with a timestamp, or use a custom name + timestamp combo. Switch modes anytime.

New

Sort into Year/Month folders

Optionally move each renamed file into a YYYY/MM/ subfolder based on its date. Existing folders are reused; missing ones are created on the fly.

Dry-run preview

See exactly what would change before a single byte moves on disk. Toggle dry-run on — it's the safe default.

Real metadata

Reads DateTimeOriginal, CreateDate, and QuickTime tags via exiftool and ffprobe. No guessing from filesystem dates.

100% local

Your files never leave your machine. No uploads, no telemetry, no accounts. Just Node and Electron talking to your disk.

Smart scanning

Walks every nested folder by default, or stays on the surface with one toggle. Already-formatted files and macOS junk (._*) are skipped automatically.

Up and running
in 30 seconds.

  1. 1

    Clone & install

    Pull the repo and install its dependencies. Node 18+ is all you need.

  2. 2

    Start the app

    npm start opens the desktop window. No build step, no bundling.

  3. 3

    Pick a folder

    Choose a mode, leave Dry run checked for the first pass, and hit Organize.

~/Projects · zsh 
$ git clone https://github.com/matheuschignolli/foldnize.git
$ cd foldnize
$ npm install
$ npm start
# → Foldnize window opens, ready to organize.

Optional: better metadata coverage

Foldnize works without any extra tools, but installing exiftool (photos) and ffmpeg (videos) unlocks original-date parsing across many more formats.

Homebrew 
$ brew install exiftool ffmpeg

Use it as a module

All of the renaming logic lives in src/file-organizer.js as a single function. Drop it into a script, a CI job, or another desktop app.

organize.js 
const { organizeFolder } = require("./src/file-organizer");

organizeFolder({
  root: "/Users/you/Pictures/2023",
  mode: "custom",                // "prefix" | "replace" | "custom"
  customName: "vacation",        // required when mode is "custom"
  organizeIntoYearMonth: true,   // move into <root>/YYYY/MM/
  scanSubfolders: true,          // recurse (default); false = top-level only
  dryRun: true,
  onLog: ({ message }) => console.log(message),
});
